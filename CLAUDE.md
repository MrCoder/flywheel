# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What Is Flywheel

A CLI-first personal work tracker with persistent memory, built for AI-assisted development workflows. It scans git history and Cursor agent transcripts to understand what you worked on, maintains a Kanban board, and remembers decisions.

## Commands

```bash
# Morning routine — scans git + transcripts, generates daily summary
bun run morning

# Task management
bun run cli task start "Task title"              # Create/move to DOING
bun run cli task done "Task title"               # Move to DONE
bun run cli task todo "Task title"               # Add to TODO
bun run cli task list                            # Show board (today)
bun run cli task list --date 2026-02-27          # Show board for specific date

# Subtasks — use --parent to nest under an existing task
bun run cli task start "Design editor" --parent "add PlantUML support"

# Add flags: --project <name>
bun run cli task start "Fix parser" --project zenuml/zenuml-core

# Activity timeline
bun run cli log "Deployed v2.1 to staging" --project zenuml/conf-app

# Memory
bun run cli remember "Using D1 for storage" --project zenuml/conf-app --tags architecture
bun run cli search "D1"                          # Search memories
bun run cli memories                             # Recent memories

# Dashboard
bun run cli generate                             # Manually regenerate dashboard HTML
bun run open                                     # Open dashboard in browser (static mode)
bun run serve                                    # Hosted mode on :3457 (serves static files + Done button API)
```

## Key Concepts

- **Date-scoped tasks** — each task belongs to a date. The board shows one day at a time.
- **Auto carry-forward** — incomplete tasks are cloned to the new day on any CLI command. Yesterday's tasks are preserved as-is for history.
- **Subtasks** — tasks can be nested under a parent with `--parent "title"`.
- **Activity timeline** — `log` command records milestones. Task start/done actions also auto-log activities.
- **Claude Code hook** — a `UserPromptSubmit` hook in `~/.claude/settings.json` auto-injects the task board into every Claude session.
- **Static dashboard** — every CLI mutation auto-regenerates `dist/*.html`. Open directly or serve via `bun run serve`.

## Architecture

```
CLI command → SQLite → generateDashboard() → dist/*.html
                                                  ↑
                              bun run serve (optional) — serves dist/ + PATCH /api/tasks/:id
```

- **CLI** (`src/cli/`) — Bun scripts, entry point is `src/cli/index.ts`. Every mutation triggers HTML regeneration.
- **Collectors** (`src/collectors/`) — git-collector, transcript-collector, file-collector, skills-collector
- **Database** (`src/db/`) — SQLite via `bun:sqlite`, stored in `data/flywheel.db`
  - `tasks` — date-scoped, with parent_id for subtasks and source_task_id for carry-forward lineage
  - `activities` — timeline events (auto-logged on task transitions + manual `log` command)
  - `memories` — persistent decisions and insights
  - `daily_summaries` — generated by morning routine
- **HTML Templates** (`src/html/`) — generates static HTML with Tailwind CSS CDN
  - `generate.ts` — orchestrator: queries DB, calls renderers, writes `dist/`
  - `layout.ts` — shared page shell, nav, escaping, time-ago utility
  - `board.ts` — task cards, date navigation, timeline
  - `memories.ts`, `daily.ts`, `skills.ts` — tab pages
- **Server** (`src/server/`) — optional Hono server on port 3457
  - Serves `dist/` as static files
  - `PATCH /api/tasks/:id` — marks tasks done (used by dashboard Done button), triggers regeneration

### Two modes

- **Static**: `bun run open` — opens `dist/index.html` directly. Read-only, no server needed.
- **Hosted**: `bun run serve` — serves the same HTML on :3457, but the Done button works (calls API, regenerates, reloads).

## Tech Stack

Bun runtime, SQLite (bun:sqlite), Hono (optional server), Tailwind CSS (CDN).
